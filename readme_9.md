This project is designed to simulate a real-world cloud data replication system where user data is copied and stored across multiple servers. In cloud environments, improper replication methods can cause serious issues such as accidental modification of original data, duplicate records, and data corruption. The objective of this program is to analyze these problems by creating nested user data structures containing dictionaries and lists, then performing replication using assignment, shallow copy, and deep copy techniques.

The program allows users to enter their roll number, and based on whether it is even or odd, a personalized mutation rule is applied. If the roll number is even, a new file is added to the replicated data, and if it is odd, an existing file is removed. Additional modifications such as changing storage usage values are also performed to simulate real replication failures. This helps demonstrate how shallow copying affects inner nested lists due to shared references, while deep copying keeps the original data safe.

After modification, the system performs an integrity analysis by detecting data leakage, checking whether deep copies remain unaffected, identifying overlapping files using sets, and analyzing mutation depth at both outer and inner levels. It also provides a before-and-after comparison to clearly show how data changes during replication.

Through this project, I learned how nested data structures work, the importance of proper data replication in cloud systems, and how Python concepts like functions, loops, conditions, sets, dictionaries, shallow copy, and deep copy can be used to solve practical data integrity problems.
